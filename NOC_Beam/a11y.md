# Accessibility

NOC_Beam runs on the on-call NOC desk. People use it tired, in the dark, at 03:00 UTC when an SBC has flipped. Accessibility here is not a checklist ‚Äî it is the difference between catching a `401 Unauthorized` loop in thirty seconds and missing it for four hours.

This file documents the floor, not the ceiling.

## Contrast

All text and signal indicators clear **WCAG 2.2 AA** at their used sizes.

| Token pair | Ratio | Used for |
| --- | --- | --- |
| `--fg-1` on `--bg-base` | 13.5:1 | primary body text |
| `--fg-2` on `--bg-base` | 8.4:1 | secondary text |
| `--fg-3` on `--bg-base` | 4.6:1 | hints, metadata (12px+ only) |
| `--beam-cyan` on `--bg-deep` | 9.7:1 | links, RX trace lines |
| `--beam-amber` on `--bg-deep` | 9.9:1 | TX trace lines, secondary highlight |
| `--success` on `--bg-base` | 8.2:1 | registered, accept |
| `--danger` on `--bg-base` | 5.1:1 | hang-up, error |

The light theme (not the default ‚Äî see `colors_and_type.css` should we add one) MUST hit the same ratios.

Color is **never the only carrier of state**. RX/TX lines in the trace are labelled `RX ‚Üê` / `TX ‚Üí` in addition to colour. Registration state shows a label (`REGISTERED`/`OFFLINE`) plus the dot. Call direction in the recent list uses a unique glyph per state (incoming/outgoing/missed) in addition to colour.

## Screen reader

Surface labels are explicit, never inferred from icon alone.

- Icon-only buttons have `aria-label`. The rail's Calls button is `aria-label="Calls"`, not relying on the visible "Calls" text below the icon (which acts as decorative reinforcement).
- The live trace drawer uses `aria-live="polite"` on the message list so new entries are announced.
- The incoming-call overlay sets `role="alertdialog"`, traps focus, and reads the caller name and SIP URI in that order.
- The `‚óè LIVE` indicator carries `aria-label="Trace is live"`.
- Registration state in the rail status pill is exposed as `aria-label="Registered to 4 of 4 accounts"`, not just visually.

## The SIP trace

The trace is the centerpiece of the product and the most accessibility-sensitive surface.

- **Selectable text.** The whole trace is real text, `user-select: text`. Never a canvas, never an image. The trace MUST be copy-pasteable into a ticket without OCR.
- **No animation on arrival** ‚Äî see `motion.md`. New messages append without movement so screen readers and low-vision users aren't ambushed.
- **Each message has a `<details>` summary** so it can be collapsed; expanded by default for sighted users, collapsed for screen reader users to skim.
- **A "Copy as ticket attachment" button** writes the trace as plain text to the clipboard.

## Type

- Minimum body size **13px** at 100% zoom. Trace lines may go to 11px but are mono and selectable ‚Äî readability is preserved at 200% zoom.
- Line-height never below 1.4 for body text.
- We do not chase tighter densities by shrinking type; we shrink padding first.

## Color modes

- **Dark by default.** This is a NOC tool; the desk is dim.
- **Light mode** (planned) MUST hit AA contrast for every pair listed above. The signal pair (cyan/amber) ports as `#0E84BF` / `#C77425` in light mode ‚Äî see the wordmark exploration cards.
- **High-contrast mode** (Windows HC) MUST be honoured. Custom colours are overridden by system colours when `(forced-colors: active)`.

## Targets

- Minimum interactive target: **28√ó28px** in dense surfaces (rail icons, dialog form controls).
- Primary action targets (Accept, Hang up, Save): **44px tall minimum**.
- Dialer keys: **52√ó52px** because they are operated by feel.

## What we won't do

- No `aria-hidden` on focusable elements.
- No `tabindex > 0`.
- No autoplay sounds outside of the ringtone (which respects the OS Do-Not-Disturb).
- No motion-driven critical feedback. If the only signal that a call is being placed is a spinner, that's a bug.
- No CAPTCHAs anywhere ‚Äî this is internal tooling.
